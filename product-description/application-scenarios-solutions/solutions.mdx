---
title: "GasStation 常用使用场景"
---


GasStation 的三大核心能力分别适用于不同的使用场景。
功能包含三类：

1. **快捷租赁（Quick Rent）**  
2. **自动租赁（Auto Rent）**  
3. **API 自动化租赁（API Rent）**

以下使用场景可用参考，具体请结合实际功能和业务需求进行操作。

---

# 1. 快捷租赁（Quick Rent）

快捷租赁适用于 **临时、偶发、低频、一次性** 的资源需求，用户无需冻结 TRX，即租即用。

---

### **场景 1：偶尔转 USDT（TRC20）但没有能量**

用户平时不常使用 TRON，当天需要转 USDT 时发现地址缺少能量，直接燃烧 TRX 会产生较高成本。

👉 使用快捷租赁，租 10～30 分钟能量即可完成一次转账。

---

### **场景 2：临时进行 DApp 操作**

用户偶尔进入某个 DApp 执行质押、兑换、合成等操作，但地址没有能量。

👉 快捷租赁补足能量后即可执行一次性合约操作。

---

### **场景 3：新钱包地址缺少基础资源**

新创建的钱包常常没有带宽或能量，基础转账可能失败。

👉 通过快捷租赁快速激活地址，让地址立即可用。

---

---

# 2. 自动租赁（Auto Rent）

自动租赁适用于 **中高频、持续性、可预测资源消耗的业务场景**。  
系统实时监控资源阈值，自动触发补充能量。

---

### **场景 1：交易所 / 托管平台提现补充能量**

热钱包每天执行大量提现操作，能量不足会导致提现失败。

👉 自动租赁在资源低于阈值时自动补充能量，保证提现流程连续稳定。

---

### **场景 2：DApp 后台地址高频调用合约**

部分 DApp 需要频繁与链交互，资源不足会影响用户体验。

👉 自动租赁持续维持后台地址资源，让合约调用保持稳定。

---

### **场景 3：批量任务执行（空投 / 返佣 / 批量转账）**

批量执行链上任务时，需要充足能量维持操作不中断。

👉 自动租赁防止因资源不足导致脚本暂停或失败。

---

---

# 3. API 自动化租赁（API Rent）

API 租赁适用于 **企业级系统集成**，如交易所、DApp 后台、清算系统等。  
系统可通过 API 全自动化触发补充能量操作。

---

### **场景 1：提现系统自动补充能量**

提现系统在执行链上转账前判断能量是否不足，不足则自动通过 API 补充能量。

👉 提现流程无需人工参与，实现全自动闭环。

---

### **场景 2：DApp 后台为用户自动补充能量（Gasless 模式）**

DApp 后端在执行智能合约前自动补充能量，让用户无需理解 Energy 概念。

👉 提升用户体验，减少因资源不足导致的失败。

---

### **场景 3：后台脚本 / Bot / 定时任务需要长时间运行**

后台机器人、清算任务或批处理脚本需要持续的链上资源支持。

👉 API 自动补充能量确保脚本在运行过程中资源始终充足。

---

---

# 总结

| 功能类型 | 最典型价值 | 关键使用场景 |
|---------|------------|--------------|
| **快捷租赁** | 即租即用、无需冻结 TRX | 单次转账 / 一次性 DApp 操作 / 新地址激活 |
| **自动租赁** | 持续补充能量、稳定业务 | 提现补充能量 / 高频 DApp / 批量任务 |
| **API 租赁** | 全自动化集成 | 提现系统 / Gasless DApp / 后台脚本运行 |

通过三种租赁方式，GasStation 能满足不同类型用户（个人、DApp、企业平台）的资源需求，从一次性操作到企业级自动化补充能量均有对应解决方案。
