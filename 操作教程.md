# 文档项目操作教程

## 📚 项目简介

这是一个使用 **Mintlify** 构建的文档项目，包含中文和英文两个版本。文档使用 `.mdx` 格式编写，导航结构通过 `mint.json` 配置文件控制。

---

## 📁 项目结构说明

```
custody-gas-doc-main 2/
├── mint.json              # 中文版配置文件（控制导航结构）
├── api-references/        # 中文版文档文件夹
│   └── gas-apis/          # API文档
│       ├── apis/          # API接口文档
│       ├── guide/         # 指南文档
│       ├── notify/        # 通知文档
│       ├── contact.mdx    # 联系页面
│       └── err-code.mdx   # 错误码文档
├── en/                    # 英文版文件夹
│   ├── mint.json          # 英文版配置文件
│   └── api-references/    # 英文版文档文件夹（结构同中文版）
├── images/                # 中文版图片资源
├── logo/                  # Logo文件
└── README.md              # 项目说明
```

---

## 🎯 核心概念

### 1. **配置文件 `mint.json`**
   - 位置：项目根目录（中文版）和 `en/` 目录（英文版）
   - 作用：控制文档的导航结构、网站外观、API配置等
   - **重要**：修改导航结构必须编辑这个文件

### 2. **文档文件 `.mdx`**
   - 位置：`api-references/` 文件夹内
   - 格式：Markdown + 特殊组件（如 `<Card>`, `<ParamField>` 等）
   - 每个文档开头有 frontmatter（用 `---` 包围），定义标题等信息

### 3. **导航结构**
   - 在 `mint.json` 的 `navigation` 字段中定义
   - 支持多级分组和嵌套

### 4. **Tabs（标签页）和 Navigation（导航）的关系** ⭐
   - `tabs` 定义网站顶部的标签页（如"API文档"、"用户指南"等）
   - `navigation` 定义侧边栏的导航菜单
   - **对应关系**：`navigation` 中的路径如果以某个 `tab` 的 `url` 开头，就会显示在那个 `tab` 下
   - 这是 Mintlify 的核心机制，用于将不同类别的文档分组到不同的标签页

## ✏️ 如何修改文档内容

### 步骤 1：找到要修改的文档文件

根据文档路径找到对应的 `.mdx` 文件：

- **中文版**：在 `api-references/` 文件夹下
- **英文版**：在 `en/api-references/` 文件夹下

**示例**：
- 如果要修改"查询余额"页面，文件路径是：
  - 中文：`api-references/gas-apis/apis/gas-balance.mdx`
  - 英文：`en/api-references/gas-apis/apis/gas-balance.mdx`

### 步骤 2：编辑文档内容

打开 `.mdx` 文件，你会看到类似这样的结构：

```mdx
---
title: 查询余额
api: GET /api/tron/gas/balance
description: "查询账户余额"
---

#### 请求参数数据结构

<ParamField query="time" type="string" required>
  当前时间戳，例: `1600128590`
</ParamField>

#### 响应参数数据结构
...
```

**修改说明**：
- `---` 之间的内容是文档元信息（frontmatter）
  - `title`：页面标题
  - `api`：API接口路径（可选）
  - `description`：描述（可选）
- `---` 之后的内容是文档正文，使用 Markdown 语法编写

### 步骤 3：保存文件

保存后，如果正在运行预览服务器，页面会自动刷新显示最新内容。

## ➕ 如何添加新文档

### 步骤 1：创建新的 `.mdx` 文件

1. 在合适的位置创建新文件，例如：
   - `api-references/gas-apis/apis/新文档名称.mdx`
   - 文件名建议使用英文小写字母和连字符，如：`new-api.mdx`

2. 编写文档内容，至少包含：

```mdx
---
title: 新文档标题
---

这里是文档内容，使用 Markdown 语法编写。

## 标题

正文内容...
```

### 步骤 2：在 `mint.json` 中添加导航项

打开 `mint.json` 文件，找到 `navigation` 部分，添加新文档的路径。

**示例**：假设你创建了 `api-references/gas-apis/apis/new-api.mdx`，想在"API接口"分组下添加：

**修改前**：
```json
{
  "group": "API接口",
  "pages": [
    "api-references/gas-apis/apis/description",
    {
      "group": "资源购买",
      "pages": [
        "api-references/gas-apis/apis/gas-balance",
        ...
      ]
    }
  ]
}
```

**修改后**：
```json
{
  "group": "API接口",
  "pages": [
    "api-references/gas-apis/apis/description",
    "api-references/gas-apis/apis/new-api",  // 新增这一行
    {
      "group": "资源购买",
      "pages": [
        "api-references/gas-apis/apis/gas-balance",
        ...
      ]
    }
  ]
}
```

**重要提示**：
- 路径不需要包含 `.mdx` 扩展名
- 路径顺序决定导航菜单中的显示顺序
- 如果同时维护中英文版本，需要在两个 `mint.json` 中都添加

### 步骤 1：从导航中移除

打开 `mint.json`，在 `navigation` 中找到对应的路径并删除。

**示例**：删除 `gas-balance` 文档

**修改前**：
```json
{
  "group": "资源购买",
  "pages": [
    "api-references/gas-apis/apis/gas-balance",  // 删除这一行
```

```json

  "group": "资源购买",
  "pages": [
    "api-references/gas-apis/apis/gas-create-order",
    ...
  ]
}
```

### 步骤 2：删除文件（可选）

如果确定不再需要该文档，可以删除对应的 `.mdx` 文件。

## 🏷️ Tabs（标签页）和 Navigation（导航）的对应关系详解

这是 Mintlify 中一个重要的概念，理解它对于管理多类别文档非常重要。

### 基本原理

**对应规则**：
- `tabs` 中的每个标签页都有一个 `url` 字段（如 `"api-references"`）
- `navigation` 中的路径如果**以某个 tab 的 url 开头**，就会显示在那个 tab 下
- 路径匹配是**前缀匹配**，即路径的前面部分必须与 tab 的 url 完全一致

### 当前项目示例（单个 Tab）

让我们看看你当前项目的配置：

```json
"tabs": [
  {
    "name": "API文档",
    "url": "api-references"    // Tab 的 URL
  }
],

"navigation": [
  {
    "group": "Gas Station",
    "pages": [
      "api-references/gas-apis/contact",           // ✅ 以 "api-references" 开头
      "api-references/gas-apis/guide/...",        // ✅ 以 "api-references" 开头
      "api-references/gas-apis/apis/description"  // ✅ 以 "api-references" 开头
    ]
  }
]
```

**对应关系**：
- 所有 `navigation` 中的路径都以 `"api-references"` 开头
- 所以它们都会显示在"API文档"这个 tab 下
- 当用户点击"API文档"标签时，侧边栏会显示这些导航项

### 多个 Tabs 的示例

假设你想添加第二个标签页，比如"用户指南"：

#### 步骤 1：创建新的文档文件夹

在项目根目录创建 `user-guide/` 文件夹，并添加一些文档：
```
user-guide/
├── getting-started.mdx
├── advanced/
│   └── tips.mdx
└── faq.mdx
```

#### 步骤 2：在 `tabs` 中添加新标签

```json
"tabs": [
  {
    "name": "API文档",
    "url": "api-references"      // 第一个 tab
  },
  {
    "name": "用户指南",           // 新增第二个 tab
    "url": "user-guide"          // Tab 的 URL
  }
]
```

#### 步骤 3：在 `navigation` 中添加对应路径

```json
"navigation": [
  {
    "group": "Gas Station",
    "pages": [
      // API 文档相关（显示在"API文档" tab 下）
      "api-references/gas-apis/contact",
      "api-references/gas-apis/guide/gas-account-guide",
      {
        "group": "API接口",
        "pages": [
          "api-references/gas-apis/apis/description",
          ...
        ]
      },
      
      // 用户指南相关（显示在"用户指南" tab 下）
      "user-guide/getting-started",        // ✅ 以 "user-guide" 开头
      {
        "group": "高级功能",
        "pages": [
          "user-guide/advanced/tips"       // ✅ 以 "user-guide" 开头
        ]
      },
      "user-guide/faq"                     // ✅ 以 "user-guide" 开头
    ]
  }
]
```

**对应关系**：
- 路径以 `"api-references"` 开头的 → 显示在"API文档" tab
- 路径以 `"user-guide"` 开头的 → 显示在"用户指南" tab
- 当用户切换 tab 时，侧边栏会自动显示对应 tab 的导航项

### 三个 Tabs 的完整示例

```json
{
  "tabs": [
    {
      "name": "API文档",
      "url": "api-references"
    },
    {
      "name": "用户指南",
      "url": "user-guide"
    },
    {
      "name": "帮助中心",
      "url": "help-center"
    }
  ],
  
  "navigation": [
    {
      "group": "Gas Station",
      "pages": [
        // ===== API文档 tab 的内容 =====
        "api-references/gas-apis/contact",
        "api-references/gas-apis/apis/description",
        
        // ===== 用户指南 tab 的内容 =====
        "user-guide/getting-started",
        "user-guide/advanced/tips",
        
        // ===== 帮助中心 tab 的内容 =====
        "help-center/contact",
        "help-center/support"
      ]
    }
  ]
}
```

### 重要注意事项

1. **路径前缀必须完全匹配**
   - ✅ 正确：tab url = `"api-references"`，路径 = `"api-references/xxx"`
   - ❌ 错误：tab url = `"api"`，路径 = `"api-references/xxx"`（不匹配）

2. **Tab 的 url 不需要斜杠**
   - ✅ 正确：`"url": "api-references"`
   - ❌ 错误：`"url": "api-references/"`

3. **Navigation 路径需要完整路径**
   - ✅ 正确：`"api-references/gas-apis/contact"`
   - ❌ 错误：`"gas-apis/contact"`（缺少 tab url 前缀）

4. **一个文档只能属于一个 tab**
   - 每个路径只能匹配一个 tab（按第一个匹配的 tab）

5. **Tab 的顺序很重要**
   - `tabs` 数组中的顺序决定标签页的显示顺序

### 如何添加新的 Tab

1. **创建文档文件夹**（如 `user-guide/`）
2. **在 `tabs` 中添加新项**：
   ```json
   {
     "name": "标签名称",
     "url": "文件夹名称"
   }
   ```
3. **在 `navigation` 中添加路径**，确保路径以 tab 的 url 开头
4. **创建对应的文档文件**

### 如何删除 Tab

1. **从 `tabs` 中删除对应的项**
2. **从 `navigation` 中删除所有以该 tab url 开头的路径**
3. **（可选）删除对应的文档文件夹**

## 🔄 如何修改文档结构（层级、分组）

文档的层级结构完全由 `mint.json` 的 `navigation` 字段控制。

### 理解导航结构

```json
"navigation": [
  {
    "group": "Gas Station",           // 一级分组
    "pages": [
      "api-references/gas-apis/contact",  // 单个页面
      {
        "group": "API接口",            // 二级分组
        "pages": [
          "api-references/gas-apis/apis/description",  // 页面
          {
            "group": "资源购买",        // 三级分组
            "pages": [
              "api-references/gas-apis/apis/gas-balance",  // 页面
              ...
            ]
          }
        ]
      }
    ]
  }
]
```

### ⭐ 为什么 `pages` 数组里可以写两种不同的东西？

这是 Mintlify 的设计特性。`pages` 数组可以同时包含**两种类型**：

#### 类型 1：字符串（单个页面）
```json
"pages": [
  "api-references/gas-apis/contact",        // ← 这是一个字符串，直接指向一个文档
  "api-references/gas-apis/err-code"        // ← 这也是一个字符串，指向另一个文档
]
```
**作用**：在导航菜单中直接显示一个可点击的页面链接

#### 类型 2：对象（分组）
```json
"pages": [
  {
    "group": "API接口",                    // ← 这是一个对象，创建一个分组
    "pages": [
      "api-references/gas-apis/apis/description"
    ]
  }
]
```
**作用**：创建一个可展开/折叠的分组，里面可以包含多个页面或子分组

#### 混合使用（你的实际配置）

看你的实际配置：

```json
"pages": [
  "api-references/gas-apis/contact",                    // ✅ 类型1：字符串（单个页面）
  "api-references/gas-apis/guide/gas-account-guide",    // ✅ 类型1：字符串（单个页面）
  {                                                      // ✅ 类型2：对象（分组）
    "group": "API接口",
    "pages": [
      "api-references/gas-apis/apis/description",
      {
        "group": "资源购买",                             // ✅ 类型2：嵌套分组
        "pages": [
          "api-references/gas-apis/apis/gas-balance"
        ]
      }
    ]
  },
  {
    "group": "交易回调通知",                            // ✅ 类型2：另一个分组
    "pages": [
      "api-references/gas-apis/notify/tron"
    ]
  },
  "api-references/gas-apis/err-code"                    // ✅ 类型1：字符串（单个页面）
]
```

**在导航菜单中的显示效果**：
```
Gas Station
├── 📄 平台介绍 (contact)              ← 字符串，直接显示
├── 📄 账户指南 (gas-account-guide)    ← 字符串，直接显示
├── 📁 API接口                         ← 对象，显示为可展开的分组
│   ├── 📄 统一说明
│   └── 📁 资源购买
│       ├── 📄 查询余额
│       └── ...
├── 📁 交易回调通知                    ← 对象，显示为可展开的分组
│   └── 📄 tron
└── 📄 错误码 (err-code)               ← 字符串，直接显示
```

### 为什么要这样设计？

1. **灵活性**：可以混合使用单个页面和分组，不需要所有内容都放在分组里
2. **层级清晰**：相关文档可以放在分组中，独立文档可以直接列出
3. **组织方便**：可以根据需要选择是否创建分组

### 实际应用示例

**场景 1：只有单个页面，不需要分组**
```json
"pages": [
  "api-references/gas-apis/contact"  // 直接列出即可
]
```

**场景 2：需要分组来组织多个相关页面**
```json
"pages": [
  {
    "group": "API接口",
    "pages": [
      "api-references/gas-apis/apis/description",
      "api-references/gas-apis/apis/gas-balance"
    ]
  }
]
```

**场景 3：混合使用（最常见）**
```json
"pages": [
  "api-references/gas-apis/contact",      // 独立页面
  {
    "group": "API接口",                   // 分组
    "pages": [...]
  },
  "api-references/gas-apis/err-code"     // 另一个独立页面
]
```

### 注意事项

1. **顺序很重要**：`pages` 数组中的顺序就是导航菜单中的显示顺序
2. **可以嵌套**：分组里可以再包含分组，形成多级结构
3. **可以混合**：同一个 `pages` 数组里可以同时有字符串和对象
4. **JSON 格式**：对象用 `{}` 包围，字符串用 `""` 包围

### 常见操作

#### 1. **调整页面顺序**

在 `pages` 数组中调整路径的顺序即可：

```json
// 修改前：gas-balance 在第一个
"pages": [
  "api-references/gas-apis/apis/gas-balance",
  "api-references/gas-apis/apis/gas-create-order"
]

// 修改后：gas-create-order 在第一个
"pages": [
  "api-references/gas-apis/apis/gas-create-order",
  "api-references/gas-apis/apis/gas-balance"
]
```

#### 2. **创建新的分组**

在 `pages` 数组中添加一个新的 `group` 对象：

```json
"pages": [
  "api-references/gas-apis/contact",
  {
    "group": "新分组名称",        // 新增分组
    "pages": [
      "api-references/gas-apis/apis/文档1",
      "api-references/gas-apis/apis/文档2"
    ]
  }
]
```

#### 3. **移动文档到其他分组**

将路径从当前位置剪切，粘贴到目标分组：

```json
// 修改前：gas-balance 在"资源购买"分组
{
  "group": "资源购买",
  "pages": [
    "api-references/gas-apis/apis/gas-balance"
  ]
}

// 修改后：gas-balance 移到"API接口"分组下
{
  "group": "API接口",
  "pages": [
    "api-references/gas-apis/apis/description",
    "api-references/gas-apis/apis/gas-balance"  // 移到这里
  ]
}
```

#### 4. **删除分组**

删除整个 `group` 对象（包括其 `pages`）：

```json
// 删除整个"资源购买"分组
// 直接删除这个对象：
{
  "group": "资源购买",
  "pages": [...]
}
```

#### 5. **重命名分组**

修改 `group` 的值：

```json
// 修改前
"group": "资源购买"

// 修改后
"group": "资源管理"
```

```json
{
```

项目包含中英文两个版本：

## 📝 文档编写格式说明

### Frontmatter（文档头部）

每个文档开头必须包含：

```mdx
---
title: 文档标题
api: GET /api/endpoint  # 可选，API文档使用
description: "描述"      # 可选
---
```

### 常用 Markdown 语法

```mdx
# 一级标题
## 二级标题
### 三级标题

**粗体文字**
*斜体文字*

- 列表项1
- 列表项2

1. 有序列表1
2. 有序列表2

[链接文字](https://example.com)

![图片描述](/images/example.png)
```

### Mintlify 特殊组件

## ⚠️ 注意事项

1. **JSON 格式**：编辑 `mint.json` 时要注意 JSON 格式正确性：
   - 使用双引号，不是单引号
   - 最后一个项目后面不能有逗号
   - 大括号和方括号要配对

2. **路径格式**：
   - 导航中的路径不需要 `.mdx` 扩展名
   - 路径使用正斜杠 `/`，不是反斜杠
   - 路径区分大小写

3. **文件命名**：
   - 建议使用小写字母、数字和连字符
   - 避免使用空格和特殊字符

4. **备份**：修改前建议备份重要文件

5. **中英文同步**：修改结构时记得同步更新两个版本

## 🆘 常见问题

### Q: 修改后页面没有显示？
A: 检查：
1. `mint.json` 的 JSON 格式是否正确
2. 文档路径是否正确（区分大小写）
3. 文件是否已保存
4. 预览服务器是否正在运行

### Q: 如何知道文档的路径？
A: 文档路径 = 文件夹路径（相对于项目根目录）+ 文件名（不含 `.mdx`）

例如：`api-references/gas-apis/apis/gas-balance.mdx` 的路径是：
`api-references/gas-apis/apis/gas-balance`

### Q: 可以修改文档的 URL 吗？
A: 文档的 URL 由文件路径决定。要改变 URL，需要：
1. 重命名或移动文件
2. 更新 `mint.json` 中的路径

### Q: 如何添加图片？
A: 
1. 将图片放到 `images/` 文件夹（中文版）或 `en/images/`（英文版）
2. 在文档中使用：`![描述](/images/图片名.png)`
